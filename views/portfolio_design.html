Website Portfolio

Designing this website, I wanted to create something that highlights my current skillset. My initial portfolio page was a one page scroll. It was originally designed with inspiration of what was the trend at the time, which was a lot of Parallax scrolling in a one page setting.

In designing this portfolio site, I wanted to further my process by incorporating CSS processors like Bootstrap, as well as actually using the classes that is used in the framework. On top of that, I wanted to create a multi-page website. It does contain a lot more info and is separated like modules. Similar to how I work in creating my full stack applications, I wanted to embody that feeling by separating my works like design, mobile applications, and web applications into separate pages.


Like most projects, some changes are required from the initial wireframe. During the creation of this portfolio site, I did make some changes in terms the design and how I thought it would make my portfolio stand out.

Skills

One of those changes that I actually removed during development is the skills page. In the design process, it seemed great at the time on how I should create a full skills page on the technology I learned and showcase my use of D3 and the bootstrap technology, but while designing it, It really just looked like logos of technologies without examples of how I used it. So I decide to scrap this page completely. It didn't do much to my goal of showcasing what I know. I feel that my addition of the skills in the about me completed that missing piece and allowed me to just list my skillset and decrease the need for websites to load a lot of images.

Contact

The second development change that I did was the contact page. I first wanted to create just a page where my social media links were showing and an email address that hooks up to a user's device email to send me an email directly to me. However, after doing a some research, I was sold into adding a form box and I think I was able to create a more welcoming vibe through it.

Challenges

In terms of challenges in designing this app, there was actually very little trouble as I had gotten more proficient in using Sketch. So being able to create the items a lot quicker without stumbling on what tools I needed. I think the biggest problem in development was actually finding the icons. There were some icons that I wanted but didn't have the technologies that I was looking for so I had to resort to using PNG files and just creating a width and height to standardize my icons.
